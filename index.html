How it works

Three steps. Zero cell signal.

MeshSignal uses the radios already in every smartphone to build an instant emergency network.

1

Send an alert

Choose type, add your GPS location and message. No data connection needed — alert is signed locally with your private key.

2

Mesh relay

Nearby devices running MeshSignal auto-receive and re-broadcast via Bluetooth LE and Wi-Fi Direct, forming a self-healing network.

3

Help arrives

Alert hops until it reaches a gateway node or a device with internet access, which forwards it to emergency services.

Features

Built for real emergencies

Every design decision was made for the moments when infrastructure fails.

Truly offline-first

No internet, no SMS, no cell signal required. Peer-to-peer only, using Bluetooth Low Energy (BLE) and Wi-Fi Direct radios in every modern phone.

End-to-end encrypted

Messages are encrypted with Curve25519. Relay nodes forward ciphertext only — they cannot read the content. Only the intended recipient decrypts.

Battery aware

Under 3% battery/hour in relay mode. Adaptive duty cycling adjusts BLE scan frequency based on detected mesh density and device battery level.

GPS + manual pin

Attaches GPS coordinates automatically. Falls back to manual location pin on an offline map tile when GPS is blocked or unavailable indoors.

Epidemic routing

Uses epidemic broadcast with TTL and seen-IDs to prevent loops. Each packet has a unique ID — devices never relay the same message twice.

Alert history

All relayed alerts are stored locally and timestamped. Full offline log available to responders and NGO coordinators after connectivity is restored.

Technology

Open protocol, auditable code

Built on proven standards — nothing proprietary in the critical path.

Transport layer BLE 5.0 + Wi-Fi Direct

Devices advertise a custom GATT service UUID. BLE handles discovery; Wi-Fi Direct handles bulk transfer for large payloads. Range: ~100m per hop outdoors.

SERVICE_UUID = "6E400001-B5A3-F393-E0A9-E50E24DCCA9E"
// BLE GATT Notification characteristic
char TX = "6E400003-..."

Routing protocol Epidemic + TTL

Each alert has a UUID, TTL (default 8), and a seen-set. Devices forward only if they haven't seen the UUID. Prevents broadcast storms on dense meshes.

// Pseudo-code relay logic
if (ttl > 0 && !seen.has(uuid)) {
seen.add(uuid)
broadcast(alert, ttl - 1)
}

Encryption Curve25519 + AES-256-GCM

Key exchange via X25519 ECDH. Message content encrypted with AES-256-GCM. Relay nodes only see destination public key and encrypted blob — zero knowledge of content.

encrypt(msg, recipientPubKey) {
const shared = x25519(myPriv, recipientPub)
return aes256gcm.seal(msg, shared)
}

Gateway nodes Raspberry Pi 4 + 4G

Low-cost Raspberry Pi 4 units with a USB 4G dongle act as internet bridges. When an alert reaches a gateway, it's POSTed to the emergency dispatch API and forwarded via SMS fallback.
